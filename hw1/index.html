Task 2: Antialiasing by Supersampling

In this section we aimed to antialias the same images from task 1 by implementing supersampling.

Antialiasing is the process of removing frequencies above the Nyquist before sampling. We can attenuate these frequencies with a 1 pixel box filter. Supersampling allows us to mimic and approximates the effect of a 1 pixel box filter by sampling multiple locations within a pixel and averaging their values, essentially antialiasing an image!

Usefullness: Supersampling is useful since it reduces the effect of “jaggies” in areas of high contrast, such as edges. As we will see, supersampling gives a more accurate representation of the geometry of an image, producing smoother lines and transitions within an image.
In this technique we essentially rasterize an image at a higher resolution then downsample to the output resolution of the frame buffer. We first needed to allocate space for a sample buffer that is our “higher resolution” image. The dimensions of this sample buffer can we defined by the width and height of the frame buffer and a supersample rate:
(width∗sample rate) x (height∗sample rate)(\text{width}*\sqrt{\text{sample rate}}) \ \text{x}\ (\text{height} * \sqrt{\text{sample rate}})(width∗sample rate​) x (height∗sample rate​)﻿
We assume the supersample rate is a perfect square. Each subsample point of this sample buffer has a color value.

Supersampling algorithm: We first compute the bounding box of the triangle ( as before). For each pixel in this box, we subdivide the pixel into sample rate number of pixels, so a pixel has grid size: (sample rate xsample rate)(\sqrt{\text{sample rate}} \ \text{x} \sqrt{\text{sample rate}})(sample rate​ xsample rate​)﻿
At each subsample point (positioned at the center of the subcell), we perform the three line test (discussed in task 1) to determine if the point is located inside the triangle. Once all subsample points of a pixel is tested, we “resolve” the supersample buffer by averaging the subsamples to compute the final pixel output. This averaging smooths the colors on the edges of the triangle, reducing “jaggies”.

Task 5: “Pixel sampling” for texture mapping 

Pixel sampling is the process of selecting pixels to represent an entire image. In texture mapping we use it to determine which texels from a texture image will be included in the final output pixel. This is necessary since a pixel’s texture coordinate doesn’t usually fall exactly on a single texel, so we must sample to estimate such value. In this task we explore 2 ways to sample such: nearest-neighbor sampling and bilinear sampling. Before sampling we first need to compute the continuous texture coordinates (u, v) from the interpolated barycentric coordinates for each. Also note that each pixel sampled is actually a subsample produced from supersampling if the supersample rate is greater than 1.


Nearest-Neighbor Sampling: this method simply finds the nearest integer texel coordinate to the continuous (u,v) coordinate and returns the color of such nearest texel coordinate. To implement this we simply round the (u, v) coordinate. This method is fast/efficient, however it creates aliased and blocky results since neighboring pixel colors are not considered. We can see this happening in the results on the right, where features, like grid lines, are actually missing and broken.


Bilinear Sampling: this method takes the 4 nearest pixels to (u,v) instead and takes a weighted average of them based on proximity to (u, v) to effectively blend their colors. This method is slower than nearest but produces smoother, more accurate images and reduces aliasing.

Zooming into the grid lines on the globe of test2.svg, using the pixel inspector, we can see how bilinear sampling clearly defeats nearest sampling. A large difference is particularly noticeable at edges with high contrast in colors, like gridlines that are thin and have distinct colors to the blue and green colors of the world map. We can see that the grid lines are broken and fragmented when using nearest sampling, but more connected when using bilinear sampling. Overall we can expect results from nearest sampling to be blocky and pixelated, while bilinear producing smoother results. However as the supersampling rate increases, bilinear sampling becomes prone to blurring.

Task 6: “Level sampling” with mipmaps for texture mapping

Level sampling is choosing a resolution/mipmap level of a texture to sample when mapping it to a surface. The idea behind this is because textures are stored at multiple resolutions in a pyramid structure.We select a level that matches the screen size of the texture best. Level sampling allows us to minimize aliasing and increase rendering efficiency since certain details that will not be seen from a distance are negated.


Implementation: for each subsample I first computed the (u,v) cotinuous texture coordinates using baycentric interpolation. Then I estimated how quickly these uv coordinates change to screen space by calculating the difference vectors in u and v directions. To do so we compare uv at (x,y) with uv at (x+1, y) and (x, y+1). We then need to scale these by the mipmap width and height to get the full resolution texture size. We then use these values to calculate the maximum derivative and take log2log_2log2​﻿ of such to calculate the continuous mipmap level. We also implemented 3 level sampling methods:

Zero level sampling: sampling from the 0th mipmap level (as in task 5)

Nearest level sampling: taking the nearest appropriate mipmap level and passing that level as a parameter to the nearest or bilinear sample function

Bilinear level sampling: computing a weighted average of the 2 adjacent mipmap levels. when we use this with bilinear pixel sampling we call it trilinear filtering

We have now implemented 3 sampling techniques: pixel sampling, level sampling, or the number of samples per pixel. Below I will discuss the tradeoffs between speed, memory usage, and antialiasing power between the three various techniques:


Pixel Sampling: Fastest speed since we only need to compute one sample per pixel. Similarly, for memory usage we only need to store one color value for each pixel, therefore it uses minimal memory. Antialiasing power is not as strong since each pixel is sampled once so fine details are not captures and jaggies effect can still be present, although minimized

Level Sampling/mipmapping: Still fast although theres a bit of overhead to compute derivatives and taking a level, but we are still sampling once per pixel. In terms of memory usage, we use more memory than pixel sampling since we need to store the mipmap pyramid. Antialiasing power is fairly strong, especially when choosing a lower resolution mipmap level.

Number of samples per pixel/Super sampling: This method is the slowest since we take multiple samples per pixel, which increases number of computations. We need a larger sample buffer to store all the subsamples until they are averaged to compute the final pixel color. Antialiasing power is strong here since it is able to average out high frequency details, producing much smoother results. Edges are smoothed, reducing jaggies.
